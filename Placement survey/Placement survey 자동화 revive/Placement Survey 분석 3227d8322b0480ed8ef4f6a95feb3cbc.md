# 1. Placement Survey의 맥락

## 1-1. 왜 이 서베이를 수행하는가?

채용 시장에는 공신력 있는 Market Share(MS) 데이터를 제공하는 기관이 없다. 우리가 시장에서 어디쯤 서 있는지를 외부 데이터로는 확인할 수 없는 구조이며, 이 공백을 메우기 위해 직접 서베이를 설계·운영하여 분기 단위로 채용 시장 내 경쟁 포지션을 추적하는 것이 Placement Survey의 존재 이유이다.

## 1-2. 무엇을 보는가?

채용 시장을 **상용직(잡코리아)**과 **임시·일용직(알바몬)** 두 축으로 분리하여, 각각의 시장에서 어떤 일이 일어나고 있는지를 파악한다. 핵심 지표는 **RMS(Relative Market Share)**이다.

**왜 MS가 아니라 RMS인가?**

구직자는 온라인 플랫폼뿐 아니라 오프라인 채널, 지인 추천, 기업 홈페이지 등 다양한 경로로 취업에 지원한다. 이 중 우리와 직접적으로 경쟁하는 **주요 온라인 플랫폼 그룹 내 상대 점유율**을 보는 것이 경쟁 역학을 파악하는 데 훨씬 유의미하기 때문이다.

**RMS 비교 대상군**

- **상용직(JK):** 잡코리아 / 사람인 / 원티드 / 리멤버
- **임시·일용직(AM):** 알바몬 / 알바천국 / 당근알바

**RMS를 보는 3단계**

RMS는 단일 숫자로 끝나지 않는다. 구직 여정의 단계별로 나누어 본다.

1. **인지 단계** — 해당 플랫폼을 알고 있는가
2. **지원 단계** — 실제로 해당 플랫폼을 통해 지원했는가
3. **재지원 단계** — 다시 이용할 의향이 있는가

이 단계별로 RMS를 추적하면, 단순히 "점유율이 올랐다/내렸다"를 넘어 퍼널 상 어디에서 이탈이 생기는지, 어디에서 강점이 있는지를 입체적으로 파악할 수 있다.

## 1-3. RMS를 어떻게 산출하는가? — Cubicle 기반 Adjusted RMS

RMS는 단순 집계로 구하지 않는다. **Cubicle이라는 세그먼트 그룹에 가중치(weight)를 적용한 Adjusted 방식**으로 산출한다.

**왜 Adjusted가 필요한가?**

분기마다 설문 응답자의 구성이 달라진다. 예를 들어 Q1에는 20대 수도권 응답자가 많고 Q2에는 30대 비수도권 응답자가 많아지면, RMS가 실제 시장 변화 없이도 응답자 구성 변화만으로 달라져 보인다. 이런 구성 효과(composition effect)를 제거하고 순수한 RMS 변화만 보기 위해 Cubicle weight의 전체 쿼터 평균값을 적용하여 보정(adjust)한다.

**Cubicle이란?**

서베이 응답자를 채널 이용 경향성이 비슷한 사람들끼리 묶은 그룹이다. 연령대, 거주지, 성별 등 여러 축(응답자 trait)을 기반으로, RMS의 추이(trend)와 절대 수치가 유사한 응답자들을 하나의 Cubicle로 grouping한다.

- JK, AM 모두 총 3개의 축을 기반으로 Cubicle을 형성
- Cubicle 자체는 이미 도출이 완료되어 있으므로, 이후 분석에서는 기존 Cubicle 구조를 그대로 사용
- 예시: "수도권에 거주하는 1020세대", "수도권에 거주하는 30세 이상" 등

**Quota 설정 (설문 1달 전 선행 업무)**

Cubicle 기반으로 Adjusted RMS를 산출하려면 각 Cubicle에 충분한 응답자가 확보되어야 한다. 따라서 매 설문 진행 약 1달 전에 Quota(각 Cubicle별 목표 응답자 수)를 확정하는 작업이 선행된다. "매 분기마다 어떤 구성의 사람들이 어느 정도 들어와야 제대로 된 RMS를 볼 수 있는가"를 결정하는 과정이다.

## 1-4. 용어 정의 (Dictionary)

분석을 시작하면 여러 생소한 용어가 보이기 때문에, 이해도 향상을 위해 용어의 의미를 숙지하는 것이 중요하다.

| 용어 | 정의 | 계산식 |
| --- | --- | --- |
| **Cubicle** | 채널 이용 경향성이 비슷한 응답자를 묶은 세그먼트 그룹. 3개 축(연령, 거주지 등)을 기반으로 RMS trend와 수치가 유사한 응답자 |  |
| **Cubicle Weight** | 전체 설문 응답자 중 특정 Cubicle에 해당하는 응답자의 비중 | (해당 쿼터 Cubicle N 응답자 수) ÷ (해당 쿼터 전체 응답자 수) |
| **Cubicle Unadjusted RMS** | 특정 Cubicle 내에서 특정 채널을 사용한 비중 (Cubicle 내 채널별 RMS) | (Cubicle N 응답자 중 Channel A 이용자 수) ÷ (Cubicle N 응답자 수) |
| **Cubicle Adjusted RMS** | Cubicle별 RMS에 weight 변동성을 보정한 값 | (특정 쿼터 Cubicle N Unadjusted RMS) × (전체 쿼터 Cubicle N weight 평균값) |
| **Total Unadjusted RMS** | 전체 표본 중 각 채널에 대한 RMS (weight 보정 없음) | Σ [Cubicle N Unadjusted RMS × 해당 쿼터 Cubicle N weight] (모든 Cubicle 합산) |
| **Total Adjusted RMS** | 쿼터별 Cubicle weight 변동에 따른 오차를 보정한 전체 RMS | Σ [Cubicle N Unadjusted RMS × 전체 쿼터 Cubicle N weight 평균값] (모든 Cubicle 합산) |

## 1-6. 최종적으로 얻고 싶은 View

> 이번 분기에 우리가 잘했는가, 못했는가?
> 

이 질문에 답하되, 전체 합산이 아니라 세그먼트별로 쪼개서 과거 분기와 비교한다.

- 어떤 세그먼트에서 우리가 강한가 / 약한가
- 각 세그먼트에서 메인 경쟁자는 누구인가
- 응답자들이 밝힌 채널 선택 이유(KBF)를 통해, 어떤 방향으로 개선하면 되겠는가

숫자 확인에서 끝나는 것이 아니라 **"그래서 어떻게 하면 되는가"라는 액션 방향까지 제시하는 것**이 이 서베이의 궁극적 목적.

## 1-7. 부가 항목

현재 상용직 서베이에는 **잡플래닛 이용 경험** 관련 문항이 추가되어 있어, 잡플래닛에 대한 사용자 인식도 함께 모니터링중.

# 2. Placement Survey 분석 업무

> **사전 참고**: 이 매뉴얼은 '기존 시트를 참고하라'는 내용이 많을 정도로, 이전 분기 분석 시트에 대한 의존도가 높다. 따라서 이전에 분석한 시트가 어떻게 구성되어 있고, 어떤 셀끼리 엮여 있고, 어떤 함수가 사용되어서 어떤 결과값이 도출되었는지를 미리 숙지하는 것이 중요하다.
> 

**전체 흐름 요약**

1. **Data Setting** — MME에서 받은 데이터를 쿼터 시트(ex. 26Q1)에 그대로 넣기
2. **Data Cleansing** — R_22Q2-25Q4에 26Q1 데이터 가져오면서 cleansing 한 번 하기 (넘버링 제거, 이유·채널·Cubicle·Seg 분류)
3. **시트별 점검** — 각 분석 시트에 새 쿼터 열 추가 + 수식 복사
4. **장표 작성** — TBU

## Step 0. Setting

**0-1. 엠브레인 Raw Data 저장**

엠브레인으로부터 받은 Survey Raw Data 파일을 공유폴더에 저장한다.

- 파일 형태: `MME_20XX년 X분기 잡코리아/알바몬 Job placement 조사 Rawdata_YYYYMMDD.xlsx`
- 파일 내부 시트 구성: `Numeric` (숫자 코드), `String` (텍스트 응답), `Variable Labels` (변수 설명), `Codebook` (코드북)
- 이 중 분석에 사용하는 시트는 `String` 시트이다 (텍스트 형태의 응답 데이터)

**0-2. 작업이 필요한 시트 구조 파악**

실질적으로 작업하는 시트와 자동 연동되는 시트를 구분해야 한다.

- **직접 작업 시트**: `1-1. 채널 Cut` (= 2-1. 채널 RMS), `1-2. Cubicle Cut` (= 2-2. Cubicle RMS)
- **자동 연동 시트**: `1. 지원 RMS` — 1-1 시트의 셀을 참조하여 자동 계산되는 구조이므로, 앞 시트의 함수를 그대로 긁어오면 된다
- **참고 시트 (수정 불필요)**: `Channel 분류표`, `이유 분류표`, `산업 분류표`, `지역 분류표`, `산업직무소득 seg`, `Cubicle_vF`

## Step 1. Raw Data Setting

> 엠브레인에서 받은 Raw Data를 분석 파일에 셋팅하는 단계이다.
> 

**1-1. 해당 쿼터의 Raw Data 시트 만들기**

분석 파일 내에 새로운 시트를 생성한다 (예: `26Q1`).

엠브레인 Raw Data 파일의 `String` 시트에서 데이터를 복사하여 이 시트에 붙여넣는다.

- String 시트의 주요 열: `no`, `B5`(인지), `B7`(지원), `B10`(재사용), `SQ1`(성별), `SQ2_2`(연령), `SQ3`(거주지), `SQ4`(직업), `SQ7`(산업), `SQ8`(직무), `SQ9`(기업규모), `SQ10`(기업유형), `SQ11`(연봉), `B9_1~3`(지원이유), `B11_1~3`(재지원이유)

## Step 2. Data Cleansing

> **R_22Q2-R25Q4 가져올 데이터 잘 맞춰서 텍스트만 남겨서 가져오고 2-1부터 2-5까지 잘 적용해 한 줄 만든 뒤에, 1499줄은 그대로 마우스로 긁어서 내리면 액셀 함수의 특성상 그대로 따라오도록**
> 

**2-0. Quarter 시트에 데이터 가져오기**

**통합 Raw Data 시트(`R_22Q2-25Q4`)에 신규 쿼터 데이터 추가**

기존 통합 Raw Data 시트의 마지막 행 아래에 새 쿼터 데이터를 추가한다.

- 시트 이름을 `R_22Q2-26Q1`으로 변경한다 (범위 업데이트 반영)
- 각 열은 다음과 같이 구성된다:

| 영역 | 열 (컬럼) | 설명 |
| --- | --- | --- |
| Raw Data | A: no, B: Quarter | 순번, 쿼터명 (예: 26Q1) — 직접 입력 |
| Raw Data | C~K: 인구통계(Segmentation 용도) | 성별, 연령, 거주지, 직업, 산업, 직무, 기업규모, 기업유형, 연봉 > 이 중 거주지, 산업, 직무, 연봉을 Segment로 사용함 |
| Raw Data | M~O: 지원이유 (원본) | B9_1, B9_2, B9_3 — 쿼터 시트에서 참조 |
| Raw Data | P~R: 지원이유 분류 | VLOOKUP으로 이유 분류표에서 카테고리 매핑 |
| Raw Data | S~U: 재지원이유 (원본) | B11_1, B11_2, B11_3 — 쿼터 시트에서 참조 |
| Raw Data | V~X: 재지원이유 분류 | VLOOKUP으로 이유 분류표에서 카테고리 매핑 |
| RMS | Y: 인지 채널 (원본), Z: 인지 채널 재분류 | B5 응답 → Channel 분류표로 재분류 |
| RMS | AA: 지원 채널 (원본), AB: 지원 채널 재분류 | B7 응답 → Channel 분류표로 재분류 |
| RMS | AC: 재사용 채널 (원본), AD: 재사용 채널 재분류 | B10 응답 → Channel 분류표로 재분류 |
| Cubicle | AE~AH: 산업Group, 산업>연령Group, 산업>연령>지역Group, 최종 Cubicle | 분류표들을 참조하여 Cubicle 할당 |
| Segment | AI~AL: 산업, 직무, 소득수준, 지역 | 산업직무소득 seg 시트에서 Seg 분류 |

쿼터 시트에서 통합 Raw Data 시트로 데이터를 가져올 때, 응답 텍스트 앞에 붙은 넘버링(예: `2) 사람인`)을 제거하고 텍스트만 남겨야 한다.

`IFERROR(TRIM(MID('26Q1'!셀위치, FIND(")", '26Q1'!셀위치) + 1, LEN('26Q1'!셀위치))),"")`

한 줄을 완성한 뒤, 해당 쿼터의 응답자 수만큼(약 1,500행) 아래로 드래그한다.

- **no.** 열: 이전 마지막 번호 이어서 순번 입력
- **Quarter** 열: 해당 쿼터명(예: 26Q1) 고정 입력
- 전체 행 선택 후 **값만 붙여넣기**(Paste Values)를 수행하여 수식을 제거하고, 시트 이름을 `26Q1`으로 변경한다

**2-1. 채널 재분류 (Channel Reclassification)**

인지·지원·재사용 채널 응답을 표준화된 채널 그룹으로 재분류한다.

`=IFERROR(VLOOKUP(원본채널셀, 'Channel 분류표'!$F:$J, 5, 0), "미분류")`

- `Channel 분류표`의 F열(채용 플랫폼 Channel)에서 원본 응답을 찾아, J열(채널 Segment)의 값을 반환
- 결과 예시: 잡코리아, 사람인, 원티드, 리멤버, 기타 온라인, 오프라인, 미분류

**2-2. 이유 재분류 (Reason Reclassification)**

지원이유·재지원이유 응답을 상위 카테고리로 재분류한다.

`=IFERROR(VLOOKUP(이유셀, '이유 분류표'!$F:$G, 2, 0), "")`

- `이유 분류표`의 F열(개별 이유 텍스트)에서 원본 응답을 찾아, G열(이유 분류 카테고리)의 값을 반환
- 결과 예시: 채용 공고, 유용한 정보 제공, 이용 편의성, 부가 서비스, 공공/제도/신뢰, 브랜드/평판, 기타

**2-3. Cubicle 분류 (Cubicle Sorting)**

각 응답자를 Cubicle에 할당한다. 3단계 분류를 거쳐 최종 Cubicle이 결정된다.

1. **산업 Group**: 산업 분류표에서 VLOOKUP → 스킬기반산업, 현장중심산업, 전문직군산업 등
2. **산업>연령 Group**: 산업 Group + 연령대 조합
3. **산업>연령>지역 Group**: 위 결과 + 지역 분류 조합 → 이것이 **최종 Cubicle**
- 최종 Cubicle 수식: `=AE열&" "&AF열&" "&AG열` (3개 축 결합)
- 사실상 이전 시트 함수를 그대로 복붙하는 것이 가장 빠르다

**2-4. Segment 분류**

심화 분석(Seg Cut, Seg 시사점)을 위해 산업·직무·소득수준·지역 Seg를 추가 분류한다.

- `산업직무소득 seg` 시트를 참조하여 VLOOKUP으로 분류

**2-5. Segment 분류**

심화 분석(Seg Cut, Seg 시사점)을 위해 산업·직무·소득수준·지역 Seg를 추가 분류한다.

**2-6. 미분류 확인**

질문이 달라지고 새로운 기타 선택지가 등장하는 등에 따라 n/a나 미분류라는 응답이 나올 경우 원인 파악 및 조치 수행 필요

**3-1. `1-1. 채널 Cut` 시트 (= 2-1. 채널 RMS)**

이 시트가 전체 분석의 핵심 엔진이다. 채널별로 3개 영역(Cubicle Adjusted RMS, Cubicle Unadjusted RMS, Cubicle Weight)이 세로로 배치되어 있다.

- **Cubicle Unadjusted RMS** (행 35~61 영역): 각 Cubicle × 쿼터별로 해당 채널을 이용한 비율을 COUNTIFS로 계산
    
    `=IFERROR(COUNTIFS('R_22Q2-25Q4'!$AB:$AB,$B$4, 'R_22Q2-25Q4'!$B:$B,D$5, 'R_22Q2-25Q4'!$AH:$AH,$C35) / COUNTIFS('R_22Q2-25Q4'!$B:$B,D$5, 'R_22Q2-25Q4'!$AH:$AH,$C35), 0)`
    
    → (해당 쿼터 & 해당 Cubicle & 해당 채널 이용자 수) ÷ (해당 쿼터 & 해당 Cubicle 전체 응답자 수)
    
- **Cubicle Weight** (행 62~88 영역): 각 Cubicle이 해당 쿼터 전체에서 차지하는 비중
    
    `=IFERROR(COUNTIFS('R_22Q2-25Q4'!$AH:$AH,$C62, 'R_22Q2-25Q4'!$B:$B,D$5) / COUNTIF('R_22Q2-25Q4'!$B:$B,D$5), 0)`
    
- **Cubicle Adjusted RMS** (행 8~34 영역): Unadjusted RMS × 전체 쿼터 평균 Weight
    
    `=IFERROR(D35 * AVERAGE($D62:$Q62), 0)`
    
    → (해당 Cubicle의 Unadjusted RMS) × (해당 Cubicle Weight의 전체 쿼터 평균)
    
- **Total Adjusted RMS** (행 6): 모든 Cubicle의 Adjusted RMS 합산 `=SUM(D$8:D$34)`
- **Total Unadjusted RMS** (행 7): `=SUMPRODUCT(Unadjusted RMS 범위, Weight 범위)`

작업 방법:

1. 잡코리아·사람인·원티드·리멤버·기타 온라인·오프라인·미분류 — 7개 채널 각각에서 26Q1 열 추가
2. 25Q4 열의 수식을 오른쪽으로 복사하여 완성
3. **AVERAGE 범위 확장 확인**: Adjusted RMS의 `AVERAGE($D62:$Q62)` 범위가 새 쿼터까지 포함하도록 조정 필요

**3-2. `1-2. Cubicle Cut` 시트 (= 2-2. Cubicle RMS)**

각 Cubicle 내에서 채널별 Unadjusted RMS를 보여주는 시트이다. 1-1 시트가 "채널 기준 → Cubicle별 분해"라면, 이 시트는 "Cubicle 기준 → 채널별 분해"이다.

- 각 Cubicle 블록 내에 잡코리아·사람인·원티드·리멤버·기타 온라인·오프라인·미분류 행이 있고, 그 아래 Cubicle Weight 행이 있다
- 수식 구조는 1-1 시트와 동일한 COUNTIFS 패턴

작업 방법: 25Q4 열 옆에 26Q1 추가 → 25Q4 수식 복사

**3-3. `1. 지원 RMS` 시트 (= 1. RMS Summary)**

이 시트는 1-1 채널 Cut 시트의 셀을 직접 참조(`='1-1. 채널 Cut'!D6`)하는 구조이므로, 1-1 작업 완료 후 수식만 옆으로 긁어오면 자동 완성된다.

- Online + Offline 전체 RMS, Online only RMS, JK/SRI/WTD/RMB only RMS 3가지 view를 제공
- Online only RMS: `=IFERROR(E7/SUM(E7:E11), "N/A")` (온라인 채널 합산 중 잡코리아 비율)
- JK SRI WTD RMB RMS: `=IFERROR(E7/SUM(E7:E10), "N/A")` (주요 4사 중 잡코리아 비율)

**3-4. `1-3. Seg Cut` 시트**

산업·직무·소득수준·지역 Segment별 Unadjusted RMS를 보여주는 시트이다.

작업 방법: 25Q4 열 옆에 26Q1 추가 → 25Q4 수식 복사

**3-5. `1-4. Seg 시사점` 시트**

Seg Cut 데이터를 기반으로 시사점을 도출하는 시트이다.

1. **온라인 플랫폼 RMS Grouping** 기준 세팅:
    - Group 1. Mainstream Seg: JK+SRI+WTD+RMB 합산 RMS 25% 이상 (H)
    - Group 2. Non-mainstream Seg: 20%~25% (M)
    - Group 3. 전문직/공기업: 20% 이하 (L)
    - Group 4. 표본 부족: Seg 표본 비중 1% 미만일 경우 분석 제외
2. Segment 표본 비중, 채널별 해당 Seg RMS 데이터 입력
3. **시사점 도출**: Overall Trend (Group별 JK RMS 시사점) + Segment Deep-dive (Seg별 JK RMS 시사점)

**3-6. `1-5. 지원 채널 선택 이유` 시트**

응답자가 해당 채널을 선택한 이유(KBF)를 분석하는 시트이다.

- 이유 카테고리: 채용 공고, 유용한 정보 제공, 이용 편의성, 부가 서비스, 공공/제도/신뢰, 브랜드/평판, 기타
- 가중 점수: 1순위 ×3 + 2순위 ×2 + 3순위 ×1로 계산
    
    `=COUNTIFS(Quarter조건, 이유분류P열=$D10)*3 + COUNTIFS(Quarter조건, 이유분류Q열=$D10)*2 + COUNTIFS(Quarter조건, 이유분류R열=$D10)*1`
    
- 전체 / 잡코리아 / 사람인 / 원티드 / 리멤버별로 분리하여 분석

작업 방법:

1. 기존 행 옆에 26Q1 추가
2. 25Q4 수식 복사 후, 참조하는 쿼터를 26Q1으로 변경
3. Top3 컬러 코딩 진행
4. 시사점: 전체/우리/경쟁사별 Top3 이유, 전Q 대비 변화, JK에게 시사하는 바

**3-7.`2. 인지 RMS` & `3. 재지원 RMS` 시트**

지원 RMS와 동일한 구조로 작업한다.

- `2. 인지 RMS` + `2-1. 채널 Cut`: 지원 RMS와 동일한 수식, 참조 열만 인지 채널(Z열)로 변경
- `3. 재지원 RMS` + `3-1. 채널 Cut` + `3-2. 재지원 채널 선택 이유`: 지원 RMS와 동일한 수식, 참조 열만 재사용 채널(AD열)로 변경

# 3. Placement Survey 장표 찍기

> 엑셀 분석이 완료되면, 결과를 PPT 장표로 정리한다. 이전 분기 장표를 복사하여 데이터와 시사점만 업데이트하는 방식으로 진행한다.

## 3-1. 장표 전체 구성

장표는 크게 4개 섹션으로 구성된다.

| No. | 섹션 | 내용 |
| --- | --- | --- |
| 01 | **Summary** | JK/AM 각각의 전체 RMS 추이, 주요 온라인 플랫폼 RMS, 인지/지원/재지원 추이, **Seg별 경쟁력 평가 매트릭스** (2x2 차트) |
| 02 | **JK Double Click** | 재지원 + 소득/산업/직무/지역 Seg별 개별 슬라이드 |
| 03 | **AM Double Click** | 재사용 + 가계소득/근무형태/직무/지역 Seg별 개별 슬라이드 |
|  | **Appendix** | Seg 분류 기준표, 채널 이유 분류표 등 참고 자료 |

## 3-2. Summary 섹션 찍는 법

### a) RMS 추이 차트 (JK 4장 + AM 4장)

JK와 AM 각각에 대해 아래 4종류의 슬라이드를 만든다.

1. **전체 채널 지원 RMS 추이** — Stacked bar chart. JK/SRI/WTD/RMB/기타온라인/오프라인/미분류 7개 채널. `1. 지원 RMS` 시트의 Online+Offline 행 데이터 사용
2. **주요 온라인 플랫폼 지원 RMS 추이** — Line chart. JK/SRI/WTD/RMB 4사. `1. 지원 RMS` 시트의 JK SRI WTD RMB 행 데이터 사용
3. **인지/지원/재지원 RMS 추이** — Line chart 3개 라인. `1. 지원 RMS`, `2. 인지 RMS`, `3. 재지원 RMS` 시트의 JK SRI WTD RMB 행에서 JK만 추출

각 슬라이드 하단에 1줄 시사점 텍스트를 넣는다.

### b) Seg별 경쟁력 평가 매트릭스 (JK 4장 + AM 4장)

각 Seg Cut(소득/산업/직무/지역)별로 1장씩 만든다. 한 슬라이드는 두 영역으로 구성:

**왼쪽: 2x2 매트릭스 (Scatter 차트)**

- X축: 과거 평균 RMS (23Q1-25Q4 또는 25Q1-25Q4, 후자는 25Q1부터 데이터만 있는 경우만)
- Y축: 최근 평균 RMS (25Q1-25Q4)
- 기준선: JK/AM 평균 RMS (과거/최근)
- 4개 사분면 해석:
    - 우상(1): 과거 이상 + 최근 이상 → 강점 유지
    - 좌상(2): 과거 이하 + 최근 이상 → 성장 중
    - 좌하(3): 과거 이하 + 최근 이하 → 최우선 개선 필요
    - 우하(4): 과거 이상 + 최근 이하 → 하락 중

**그래프 실제로 찍는 방법**

- 모든 Seg에 대해 과거와 최근 평균 RMS를 구해서 Scatter 그래프에 찍기
    - 1-3. Seg Cut의 [그래프]로 숨겨져 있는 항목 열어서 26Q1 넣고 늘린 뒤에 확인하기
- 4개 사분면 분류 진행하기

**오른쪽: Seg별 세부 요약 테이블**

- 열: Seg명 / 주요 경쟁자 / 채널 개선 방향성
- 각 사분면별로 그룹화하여 정리
- 주요 경쟁자와 채널 개선 방향성은 1-3. Seg Cut의 [그래프]에서 제일 높은 RMS를 갖고 있는 경쟁자, 그들이 우리 대비 잘한다고 뽑힌 이유를 뽑아서 넣기

## 3-3. Double Click 섹션 찍는 법

Summary에서 도출된 각 Seg를 개별적으로 Deep-dive하는 슬라이드이다. **재지원 1장 + Seg Cut별 전체 슬라이드**로 구성된다.

### a) 재지원 RMS Double Click (1장)

- 경쟁사별 재지원 채널 선택 이유 Top3를 Donut chart로 표시
- 이유 카테고리 레이블 표시 (JK: 채용공고/유용한정보제공/이용편의성/브랜드평판/부가서비스, AM: 아르바이트공고/유용한정보제공/이용편의성/사용자경험및인지도/부가서비스)
- 데이터 출처: `3-2. 재지원 채널 선택 이유` 시트

### b) Seg별 Double Click (각 Seg당 1장)

모든 Seg별 Double Click 슬라이드는 **동일한 포맷**을 따른다:

**슬라이드 레이아웃**

- **좌측**: 해당 Seg 주요 경쟁사 RMS 추이 (Line chart)
    - JK: JK/SRI/RMB/WTD 4개 라인
    - AM: AM/AH/DG 3개 라인
- **우측**: 해당 Seg 주요 경쟁사 채널 선택 이유 Top3 (Donut chart)
    - 경쟁사별로 각각 Donut chart 1개씩
    - 이유 카테고리 레이블은 슬라이드 상단에 공통으로 표시
- **하단**: 시사점 텍스트 (1~2줄)
    - 1줄: RMS 추이에서 읽히는 경쟁 현황
    - 2줄: 경쟁사 사용자 탈취를 위한 개선 방향

**데이터 출처**

- RMS 추이 차트: `1-3. Seg Cut` 또는 `1-1. 채널 Cut` 시트에서 해당 Seg를 필터링하여 추출
- 채널 선택 이유: `1-5. 지원 채널 선택 이유` 시트에서 해당 Seg + 해당 채널 필터링
- 시사점: RMS 추이와 KBF 분석을 종합하여 직접 작성

### c) 시사점 작성 가이드

모든 Double Click 슬라이드의 시사점은 다음 패턴을 따른다:

- **1줄 (경쟁 현황)**: `[기간] [RMS 동향 설명]` — 예: "23Q1-25Q4 SRI와 치열하게 경쟁 중이며, 최근 격차를 벌리는 추세"
